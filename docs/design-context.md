# 设计语境 — FinderSnap

## 用户

- **目标用户**：普通 macOS 用户（而非开发者或高级用户）
- **使用场景**：日常工作中频繁打开 Finder 窗口，希望窗口自动出现在期望位置和大小，减少手动调整
- **经验水平**：对 macOS 有一定了解，但不一定熟悉 Accessibility 权限或菜单栏应用的运作方式
- **时间预期**：希望"安装即用"，不愿花过多时间学习配置

## 品牌个性

**极简 / 专业 / 静默**

- 像系统原生工具一样融入 macOS，不抢注意力
- 用户感受到的是系统被智能优化了，而不是"多了一个 app"
- 静默工作、无需干预，是 FinderSnap 的核心体验承诺

**语气**：克制、清晰、不解释显而易见的事。每个 UI 元素都要有用。

**反面参照**：
- 不应该像营销型 SaaS 产品的夸张引导（大插画、彩虹渐变、"欢迎来到..."式弹窗）
- 不应该像游戏化的新手教程（徽章、进度条、强制点击）
- 不应该有过度动画或视觉噪音

## 美学方向

- **严格遵循 macOS 原生设计**：使用系统默认控件、SF Symbols、标准间距、Apple 的语义颜色
- 视觉上与系统偏好设置保持一致，让用户感到熟悉和可信
- 动画保持克制，使用系统标准过渡（sheet、fade），不添加自定义特效
- 颜色仅使用系统语义色（.accentColor、.secondary、.green 等），不使用自定义品牌色
- 布局遵循 macOS Human Interface Guidelines 的表单/设置面板惯例

## 设计原则

1. **静默优先**：Onboarding 不应该是"仪式"，而是让用户尽快到达 aha moment（打开 Finder 窗口看到自动调整）。引导流程应尽可能短，不超过 3 步。

2. **权限引导必须手把手**：Accessibility 权限是最大流失点。必须明确告诉用户去哪里设置、点击什么、为什么需要，最好有截图或系统设置直接跳转。

3. **可见性 > 发现性**：LSUIElement 应用启动后没有 Dock 图标，用户可能以为没打开。Onboarding 必须让用户明确知道应用已在菜单栏运行。

4. **上下文教学**：不在首次启动时教一切。只在用户需要某个功能时才提示（例如首次看到"Only effect to first Finder's window"这样的选项不需要解释，但权限设置必须解释）。

5. **允许跳过**：提供明确的"跳过"或"以后再说"选项。尊重用户想自己探索的意愿。

6. **一次到位**：Onboarding 只展示一次。用 UserDefaults 标记已完成，绝不重复打扰用户。

## 实现说明

Onboarding 采用 3 步流程实现（`OnboardingView.swift` + `OnboardingWindowController.swift`）：

1. **欢迎** — 说明 FinderSnap 在菜单栏运行；展示应用图标，让用户知道该找什么
2. **权限** — 通过 `AXUtils.trustPublisher` 实时监听权限状态；提供「授予辅助功能权限」按钮和「打开辅助功能设置」链接；授权后显示勾选标记
3. **快速设置** — 预览推荐默认值（调整大小 1200×800、在主屏幕居中、0.25 秒平滑动画）；「使用推荐设置」一键应用

所有步骤均遵循上述设计原则：每步均可跳过、关闭窗口即视为完成 onboarding、最后一步可直接打开设置。
